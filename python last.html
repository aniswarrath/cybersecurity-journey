Error Handling — try · except · finally · raise

Why Error Handling Exists

When your program runs — things can go wrong. User types text instead of a number. File doesn't exist. Division by zero. Without error handling — Python crashes and shows an ugly error message. With error handling — you catch the problem and respond gracefully. The program continues running instead of crashing.

Q: If if/elif/else is sufficient — why does try/except exist?
A: if/elif/else checks conditions BEFORE code runs — things you anticipate. try/except catches crashes DURING code running — things you cannot predict in advance. You cannot write an if statement to handle every possible wrong input a user might type. try/except handles anything that goes wrong at runtime — no matter what it is.

The Complete Structure

try:
    # code that might fail goes here
except ValueError:
    # runs if ValueError occurs
except ZeroDivisionError:
    # runs if ZeroDivisionError occurs
except Exception as e:
    # catches ANY other error — e tells you what went wrong
    print(f"Unexpected error: {e}")
else:
    # runs ONLY if NO error occurred — no exception was raised
finally:
    # runs ALWAYS — error or no error — used for cleanup

finally — What It Is and When to Use It

finally runs NO MATTER WHAT — whether an error occurred, whether the error was caught, whether the code succeeded. It always executes. Most important use — cleanup — closing files, closing sockets, releasing resources.

# Without finally — socket might stay open if error occurs
sock = socket.socket()
try:
    result = sock.connect_ex((ip, port))
except Exception as e:
    print(f"Error: {e}")
    # if we crash here — sock.close() never called — resource leak
sock.close()

# With finally — socket ALWAYS closes no matter what
sock = socket.socket()
try:
    result = sock.connect_ex((ip, port))
except Exception as e:
    print(f"Error: {e}")
finally:
    sock.close()  # always runs — always closes — no resource leak

else in try/except

else runs ONLY if no exception was raised in the try block. Different from finally — finally always runs, else only runs on success.

try:
    number = int(input("Enter number: "))
except ValueError:
    print("Not a number")
else:
    print("Valid number entered")   # only runs if no ValueError
finally:
    print("Always runs")            # runs regardless

raise — Manually Trigger an Error

raise lets you trigger an error yourself — useful when you want to treat a business rule violation like an error.

def check_username(username):
    if username[0].isdigit():
        raise ValueError("Username cannot start with a number")
    if len(username) < 3:
        raise ValueError("Username too short — minimum 3 characters")
    return True

try:
    check_username("123anish")
except ValueError as e:
    print(f"Invalid username: {e}")
Q: How do developers know what errors to handle?
A: Real workflow — Write code without try/except first — happy path only. Run it and break it intentionally — type wrong inputs, use bad files. Python shows the error name — ZeroDivisionError: division by zero. Copy that error name into your except clause. Repeat until all cases handled. Nobody memorizes everything — Python tells you what went wrong.

with open() as f — Context Manager

The with statement is the correct way to open files. It automatically closes the file when the with block ends — even if an error occurs. You never need to call f.close() manually.

# Without with — must close manually — easy to forget
f = open("log.txt", "w")
f.write("Aniswar Rath\n")
f.close()   # if error happens before this — file never closes

# With with — closes automatically — always safe
with open("log.txt", "w") as f:
    f.write("Aniswar Rath\n")
# file automatically closed here — even if error occurred inside

Writing to a File — "w" mode

with open("log.txt", "w") as f:
    f.write("Aniswar Rath\n")    # \n = newline — starts new line
    f.write("May 14, 2026\n")
"\n" — newline character: Without \n — everything writes on one line. With \n — each write starts on a new line. Like pressing Enter on your keyboard.

Reading from a File — "r" mode

with open("log.txt", "r") as f:
    content = f.read()    # reads entire file as one string
    print(content)        # must print to see it

# Store in variable — use outside the with block
with open("log.txt", "r") as f:
    content = f.read()

print(content)        # still works — file closed but content saved in variable
print(len(content))   # can use content anywhere after reading
Q: Why store in content = f.read() instead of just print(f.read())?
A: You CAN print directly — print(f.read()) works. But storing in content lets you use the data multiple times and outside the with block. Once with block ends — file is closed. content variable still holds the data — you can print it, modify it, search it, write it elsewhere. Direct print — data is gone after printing.

Appending to a File — "a" mode

# First write — creates file
with open("log.txt", "w") as f:

    f.write("May 14, 2026\n")

# Append — adds to end — does not overwrite
with open("log.txt", "a") as f:

with open("log.txt", "r") as f:

Read and Write — "r+" mode

r+ opens file for both reading and writing. File must already exist. Important: reading moves the cursor to the end — writing then happens at the end — use f.seek(0) to rewind cursor to beginning before reading again.

with open("log.txt", "r+") as f:
    content = f.read()        # read existing content — cursor moves to END
    f.write("\nNew line")     # write at cursor position — end of file
    f.seek(0)                 # rewind cursor to BEGINNING of file
    updated = f.read()        # read everything including new line
    print(updated)            # shows all content including new line

f.seek() — Moving the File Cursor

Think of a file like a cassette tape. Reading and writing moves a playhead along the tape. f.seek() moves the playhead to any position.

Sockets — Python Network Programming

What a Socket Is

A socket is an endpoint for sending and receiving data over a network. Every network connection ever made — HTTP, SSH, FTP, SMTP — uses sockets underneath. When your browser visits google.com — it creates a socket, connects to Google's IP on port 443, sends an HTTP request, receives the response, closes the socket. A socket is the fundamental building block of all network communication.

Q: What is a socket in simple terms?
A: Think of it like a telephone. You need a phone on each end to have a conversation. Socket = the phone. IP address = phone number. Port = extension number. connect_ex() = dialing. send/recv = talking. close() = hanging up. Both sides need a socket — your program creates one, the server already has one listening.

settimeout — Why It Is Critical

Without timeout — scanner waits forever on filtered ports — ports where firewall silently drops packets. A scan of 1000 ports could take hours. With 1 second timeout — maximum 1000 seconds worst case. In practice much faster — most ports respond in milliseconds.

sock.settimeout(1)    # wait maximum 1 second — then give up and move on

check_port Function — Complete with Explanation

import socket

def check_port(ip, port):
    try:
        # Step 1 — create TCP/IPv4 socket — pick up the phone
        sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)

        # Step 2 — set timeout — don't wait more than 1 second
        sock.settimeout(1)

        # Step 3 — try to connect — dial the number
        result = sock.connect_ex((ip, port))

        # Step 4 — close socket — hang up
        sock.close()

        # Step 5 — return result
        # result == 0 means connection succeeded — port OPEN
        # result != 0 means connection failed — port CLOSED or FILTERED
        return result == 0

    except Exception as e:
        return False   # any unexpected error — treat as closed

What connect_ex Does Under the Hood

When you call connect_ex() — Python automatically performs the full TCP three-way handshake — SYN, SYN-ACK, ACK — that you already understand from networking theory. If the handshake completes — result = 0 — port is open. If server sends RST — Reset — port closed — non-zero result. If no response within timeout — filtered port — timeout error.

Why Library Network Blocked Socket Connections

When testing at the library — all ports returned False including Google's port 80 and 443. This is called egress filtering — the library's firewall blocks outgoing connections to certain ports. This is a legitimate security control used in organizations to prevent data exfiltration and control internet access. Your code was correct — the network was the variable. At home — ports 80 and 443 on Google return True.

Real security lesson: Finding that an organization blocks outgoing connections on certain ports is a valid penetration test finding. It tells you about their security posture — what they allow and what they restrict. Always test on your home network when library network blocks results.

Complete Port Scanner — Build Target

import socket
from datetime import datetime

def check_port(ip, port):
    try:
        sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
        sock.settimeout(1)
        result = sock.connect_ex((ip, port))
        sock.close()
        return result == 0
    except:
        return False

def scan_target(ip, ports):
    print(f"\n{'='*40}")
    print(f"Target: {ip}")
    print(f"Scan started: {datetime.now().strftime('%H:%M:%S')}")
    print(f"{'='*40}")

    open_ports = []

    for port in ports:
        if check_port(ip, port):
            print(f"Port {port} — OPEN")
            open_ports.append(port)
        else:
            print(f"Port {port} — closed")

    print(f"\nScan complete — {len(open_ports)} open ports found")
    return open_ports

# Common ports to scan
common_ports = [21, 22, 23, 25, 53, 80, 443, 3306, 8080, 8443]

# Get target from user
target = input("Enter target IP or domain: ")

try:
    ip = socket.gethostbyname(target)
except socket.gaierror:
    print("Could not resolve target")
    exit()

# Run scan
open_ports = scan_target(ip, common_ports)

# Save results to file
with open("scan_results.txt", "w") as f:
    f.write(f"Scan results for {target} ({ip})\n")
    f.write(f"{'='*40}\n")
    for port in open_ports:
        f.write(f"Port {port} — OPEN\n")
    f.write(f"\nTotal: {len(open_ports)} open ports\n")

print("Results saved to scan_results.txt")
