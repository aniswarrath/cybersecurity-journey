Part 19 — While Loops Complete

Basic While Loop

Runs as long as a condition is True. Use when you don't know how many times to repeat.

PartWhat it doesWhy it matters
count = 0Initialize variable before loopMust exist before while checks it — else NameError
while count < 5Check condition each iterationIf True — run block. If False — stop.
count += 1Update variable each loopCRITICAL — without this loop runs forever
Infinite loop: Forgetting to update the variable means condition never becomes False. Python loops forever. Press Ctrl+C to escape.

while True Pattern

Runs forever until a break statement exits it. Used when number of iterations is unknown.

CodeWhat it does
while True:Loop runs forever — only exits via break
attempts += 1Count each iteration
if correct: breakExit when condition met
if attempts >= 3: breakSafety exit — prevents infinite loop
Real world: Every login system — keep asking for password until correct OR max attempts reached. Banks allow 3 PIN attempts then block card.

Part 22 — Git and GitHub

Daily Git Workflow — Memorize This

StepCommandWhat it does
1cd ~/cybersecurity/pythonNavigate to your project folder
2git add .Stage ALL changed files
3git commit -m "what you built"Save snapshot with description

git add . vs git add filename: git add . stages everything. git add filename.py stages one specific file. Use git add . for daily commits.
Commit message rule: Describe what you built — "Add login authentication system" not "edited week1.py".
File organization: Use separate files for each concept — week1.py, practice.py, functions.py. Never delete old code — create new files instead. Old code stays on GitHub even if you delete locally.

Part 23 — Functions

What is a Function

A reusable block of code. Define once — use many times. Avoids repeating the same code.

Function Anatomy

PartWhat it isExample
defKeyword — starts function definitiondef greet
function_nameYou choose — descriptive, lowercase, underscoresgreet, check_password, scan_port
(parameter)Input function receives — like a variable(name), (port), (password)
: indented blockWhat the function does — must be indented 4 spacesprint, calculate, return

print vs return

Whatprint inside functionreturn
Shows resultYes — on screenNo — sends back to caller
Can reuse resultNo — gone after displayYes — store in variable
AnalogySaying answer out loud — heard once, goneWriting answer on paper — keep and use later

Early Return Pattern

Return exits function immediately. No need for else when return is used.

With elseWith early return — cleaner
if x: return "A" else: return "B"if x: return "A" · return "B"

Function Call Order

StepWhatWhy
1Define function with defTeach Python what the function does
2Collect input if neededGet data from user
3Call function with that dataExecute the function
Must define before calling. If you call a function before defining it — Python says NameError — it doesn't know what it is yet.
